# 驱动规格说明

## 1. GPIO 驱动

### 功能概述
- 基于寄存器直接操作，不依赖 HAL 库
- 支持 STM32F103 和 ESP32 双平台
- 提供引脚初始化、读写、中断配置功能

### 通信稳定性优化
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 引脚电平不稳 | 时钟未使能 | 写入前检查时钟并等待稳定 |
| 输出切换延迟 | BSRR/BRR寄存器竞态 | 使用原子操作写入 |
| 输入误判 | 引脚浮空 | 配置内部上拉电阻 |

### API
```c
int  gpio_init(uint8_t port, uint8_t pin, uint8_t mode);
int  gpio_write(uint8_t port, uint8_t pin, uint8_t level);
int  gpio_read(uint8_t port, uint8_t pin);
int  gpio_init_peripherals(void);
```

## 2. I2C 驱动

### 功能概述
- 支持 Standard Mode (100KHz) 和 Fast Mode (400KHz)
- 精确时钟分频配置
- ACK/NACK 检测与重试机制
- 总线仲裁丢失恢复

### 时序优化细节
```
Fast Mode 时序参数 (400KHz, PCLK1=36MHz):
  CCR = PCLK1 / (25 * 400K) ≈ 3.6 → 取4
  TRISE = (300ns / 27.7ns) + 1 ≈ 11
  实际SCL频率 ≈ 400KHz ± 2%
```

### 常见问题排查
| 问题 | 现象 | 排查方法 | 解决方案 |
|------|------|----------|----------|
| I2C时序偏移 | 数据读取不稳定 | 示波器测量SCL频率/占空比 | 调整CCR分频、TRISE参数 |
| 从机无应答 | NACK持续触发 | 逻辑分析仪抓取地址字节 | 检查从机地址、供电、上拉电阻 |
| 总线锁死 | SDA持续低电平 | 示波器观察总线状态 | 软件复位I2C外设，发送9个SCL脉冲恢复 |

### 调试工具使用
1. **示波器**: 测量SCL频率、占空比，验证时序参数
2. **逻辑分析仪**: 解析I2C报文，定位ACK/NACK、数据错误

## 3. SPI 驱动

### 功能概述
- 全双工主机模式 (Mode0: CPOL=0, CPHA=0)
- 自适应波特率分频 (PCLK/2 ~ PCLK/256)
- CRC 校验支持

### 数据丢包解决方案
| 原因 | 检测方法 | 修复措施 |
|------|----------|----------|
| 时钟速率过高 | 示波器观察波形失真 | 降低SPI速率分频 |
| CS时序不足 | 逻辑分析仪测CS-to-CLK间隔 | 增加CS建立/保持时间 |
| 从机响应慢 | 读取数据全为0xFF | 增加传输间隔延时 |

## 4. UART 驱动

### 功能概述
- 8N1 帧格式 (8数据位, 无校验, 1停止位)
- 波特率精确计算 (小数分频)
- 帧错误/噪声/溢出检测

### 串口乱码排查
```
波特率偏差计算:
  目标: 115200 bps, PCLK=72MHz
  DIV = 72000000 / (16 * 115200) = 39.0625
  整数部分 = 39, 小数部分 = 1/16 = 0.0625
  实际波特率 = 72000000 / (16 * 39.0625) = 115200 (精确)
  偏差 > 2% → 调整PCLK或使用不同晶振
```

### 外设失联恢复机制
1. 检测连续3次接收超时 → 判定设备离线
2. 重新初始化 UART 外设
3. 发送 AT 测试指令验证恢复
4. 仍失败 → 上报设备离线告警
